# Phase 15 — Plan 01: Operational Logging (Panic Recovery)

**Requirements:** OPS-01
**Depends on:** Nothing
**Estimated tasks:** 1
**Risk:** Low — safety net middleware, no behavioral change to normal paths

## Changes

### 1. Add `panicRecoveryMiddleware` in `main.go`

- Uses `defer` + `recover()` to catch panics from the entire handler chain
- Logs the panic value and full stack trace (`debug.Stack()`) via `log.ErrorContext`
- Returns HTTP 500 to the client
- Wraps the entire middleware chain: `panicRecoveryMiddleware(log, loggingMiddleware(log, handler))`

### Middleware Order

```
panicRecoveryMiddleware   ← outermost (catches panics from any layer)
  └── loggingMiddleware   ← request logging + correlation ID
       └── connect.New()  ← ServeMux with handlers
```
