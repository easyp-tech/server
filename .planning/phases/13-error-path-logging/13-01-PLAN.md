**Depends on:** Phase 12 (for request_id context propagation)
**Estimated tasks:** 1
**Risk:** Low ‚Äî isolated to `commits.go`, no behavioral change

## Changes

### 1. Add `logHandlerError` helper on `commitServiceHandler`

Centralized helper that:
- Determines `protocol` from URL path (`"v1"` or `"v1beta1"`)
- Extracts `request_id` from context
- Accepts extra structured attrs (`owner`, `repo`, etc.)
- Logs at WARN for 4xx, ERROR for 5xx
- Calls `http.Error` after logging

### 2. Replace all `http.Error` calls with `h.logHandlerError`

Every error return path in ServeHTTP, ServeGraph, ServeDownload, ServeGetModules now logs structured diagnostic info before writing the HTTP response.

### 3. Consistent attribute naming (ERR-05)

All handler-level error logs use: `protocol`, `request_id`, `error`, `status`, plus per-handler context (`owner`, `repo`, `commit`).
